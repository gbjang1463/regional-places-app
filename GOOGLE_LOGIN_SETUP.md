# 구글 로그인 설정 가이드

이 앱은 Google Identity Services (GIS) 기반 클라이언트 사이드 OAuth를 사용합니다. 별도 백엔드 없이 GitHub Pages에서 바로 동작합니다.

## 1. Google Cloud 프로젝트 만들기

1. <https://console.cloud.google.com/> 접속
2. 우측 상단 **프로젝트 선택 → 새 프로젝트**
3. 프로젝트 이름: 예) `regional-places-app`

## 2. OAuth 동의 화면 구성

1. 좌측 메뉴 → **API 및 서비스 → OAuth 동의 화면**
2. **외부(External)** 선택 → 만들기
3. 필수 입력만 채우기:
   - 앱 이름: `지역별 장소 앱`
   - 사용자 지원 이메일: 본인 Gmail
   - 개발자 연락처 정보: 본인 Gmail
4. **저장 후 계속** → 범위(Scopes) 단계 그대로 통과 → **테스트 사용자**에 본인 Gmail 추가
5. 저장

## 3. OAuth 클라이언트 ID 발급

1. **API 및 서비스 → 사용자 인증 정보**
2. **+ 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
3. 애플리케이션 유형: **웹 애플리케이션**
4. 이름: 예) `web-prod`
5. **승인된 JavaScript 출처**에 추가:
   - `https://gbjang1463.github.io`
   - `http://localhost` (로컬 테스트용, 선택)
6. **승인된 리디렉션 URI**: 비워둠 (GIS 버튼 방식은 불필요)
7. **만들기** 클릭

## 4. 클라이언트 ID 적용 (2가지 방법 중 택1)

### 방법 A — 코드에 하드코딩 (모든 사용자에게 적용)

`index.html` 내부 JS의 `DEFAULT_CLIENT_ID` 값을 발급받은 ID로 교체 후 푸시.

```js
const DEFAULT_CLIENT_ID = '123456789-abcdef.apps.googleusercontent.com';
```

### 방법 B — 브라우저에서 직접 입력 (코드 수정 없음)

1. 앱 접속 → 우측 상단 **⚙️ 구글 로그인 설정 필요** 클릭
2. 발급받은 Client ID 붙여넣기 → **저장**
3. localStorage에 저장되어 같은 브라우저에서는 유지됨

> 첫 테스트 단계라면 **방법 B**가 빠릅니다. 모두에게 공개하려면 **방법 A**로 코드에 반영.

## 5. 동작 확인

1. 앱 새로고침 → "Sign in with Google" 버튼이 헤더 우측에 노출됨
2. 클릭 → 구글 계정 선택 팝업
3. 로그인 성공 시 → 헤더에 프로필 사진·이름·이메일 표시
4. 우측 패널 상단에 환영 메시지 노출
5. **로그아웃** 버튼으로 세션 종료

## 6. 보안 및 한계

- 이 방식은 **클라이언트 사이드 인증**만 수행합니다. 사용자의 신원을 확인할 수는 있지만 서버 보호 자원에는 접근할 수 없습니다 (현재 앱은 서버가 없어 무관).
- ID 토큰(JWT)은 localStorage에 저장됩니다. XSS 노출 시 토큰이 유출될 수 있으므로 운영 단계에선 보안 검토 필요.
- 같은 브라우저에서 자동 로그인이 되지 않게 하려면 GIS의 `auto_select: false` 옵션을 유지하세요 (현재 그렇게 설정됨).

## 7. 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 버튼이 안 보임 | Client ID가 비어있음 → 4단계 적용 |
| `idpiframe_initialization_failed` | JavaScript 출처에 현재 도메인 누락 → 3-5 단계 확인 |
| `redirect_uri_mismatch` | GIS 버튼 방식엔 해당 없음. 다른 OAuth 흐름과 혼동 가능 |
| 팝업이 차단됨 | 브라우저 팝업 허용 |
| 본인 계정만 로그인됨 (외부 차단) | 2단계의 **테스트 사용자** 목록에 추가 또는 앱 게시 |
