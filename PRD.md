## 1. 배경 (Background)

내부적으로 관리하던 `메모.xlsx` 형식의 지역별 장소 리스트(도/지역/이름)는 단순 텍스트라 공유·재사용이 어렵고, 장소를 선택했을 때 동선·맛집·안전정보 같은 후속 컨텍스트가 분리된 노트에 흩어져 있다. 이를 **한 화면에서 장소 탐색 + 다관점 분석**을 제공하는 단일 페이지 앱으로 통합한다.

## 2. 목표 (Goals)

### 2.1 비즈니스 목표
- 비공식적으로 모아온 장소 메모를 **재사용 가능한 디지털 자산**으로 전환
- 신규 사용자가 가입·설치 없이 **3초 안에** 앱을 사용할 수 있도록 한다
- 향후 협업·공유 기능으로 확장 가능한 베이스 라인 확보

### 2.2 사용자 목표
- 도/지역 단위로 빠르게 장소를 훑어볼 수 있다
- 장소 선택 시 "가이드 한 명이 옆에 있는 듯한" 즉답형 정보를 얻는다 (동선·로컬·안전)

### 2.3 비목표 (Non-goals)
- 실시간 지도/길찾기 통합 (Phase 2 이후 검토)
- 사용자 인증·계정 시스템 (Phase 2 이후 검토)
- 모바일 네이티브 앱 패키징 (현재는 반응형 웹으로 충분)

## 3. 사용자 (Users & Personas)

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| **P1. 주말 여행 기획자** | 30~40대, 가족 단위 근거리 여행을 자주 계획 | 검증된 장소 빠르게 훑기, 동선 짜기 |
| **P2. 로컬 탐색가** | 20~30대, 평일 저녁/주말 새 장소 발굴 선호 | 맛집·특색 등 로컬 인사이트 |
| **P3. 안전 우선 가족** | 어린이 동반, 사고 리스크 사전 점검 필요 | 수심·날씨·통제 여부 등 안전 정보 |

## 4. 사용자 시나리오 (User Stories)

1. **US-1**: 사용자로서, 도/지역 필터로 후보 장소를 좁혀서 보고 싶다.
2. **US-2**: 사용자로서, 특정 장소를 누르면 동선·일정 추천을 받고 싶다.
3. **US-3**: 사용자로서, 같은 장소에 대해 로컬 가이드의 시각으로도 정보를 보고 싶다.
4. **US-4**: 사용자로서, 안전상 주의사항을 사전에 확인하고 싶다.
5. **US-5**: 사용자로서, 모바일/태블릿/PC 어디서나 동일하게 사용하고 싶다.
6. **US-6** (Phase 2): 사용자로서, 장소를 즐겨찾기에 추가하고 다음 방문 때 다시 보고 싶다.
7. **US-7** (Phase 2): 사용자로서, 새 장소를 직접 추가·편집하고 싶다.

## 6. 비기능 요구사항 (Non-Functional Requirements)

- **성능**: 초기 로드 < 1초 (단일 HTML, 외부 의존성 0)
- **호환성**: 크롬·엣지·사파리 최신 2버전, iOS Safari / Android Chrome 정상 동작
- **접근성**: 키보드 포커스 가능, 최소 명도 대비 WCAG AA 수준
- **유지보수**: 데이터(`PLACES` 배열) 1군데 수정만으로 화면 반영
- **개인정보**: 백엔드 없음 — 사용자 데이터 수집·전송 없음

## 7. 데이터 모델

```ts
type Place = {
  province: string;   // '도' (예: '충북', '경기도')
  region: string;     // '시·군·구' (예: '도대리', '포천')
  name: string;       // 장소명 (예: '용소폭포')
  tag: string;        // 카테고리 (예: '폭포·자연')
};

type Agent = {
  id: 'planner' | 'local' | 'safety';
  icon: string;
  name: string;
  role: string;
  advise: (p: Place) => string;
};
```

## 8. UI/UX 요구사항

- 2단 그리드 (장소 패널 1.1 : 에이전트 패널 1.0), 모바일에서는 1단으로 스택
- 카드 hover시 1px 상승 + 강조색 보더
- 선택된 장소는 강조색 보더 + 약한 배경 하이라이트
- 에이전트 탭은 토글, 같은 장소를 여러 관점으로 비교 가능
- 컬러 토큰 7종(`--bg`, `--panel`, `--panel-2`, `--line`, `--text`, `--muted`, `--accent`/`--accent-2`)으로 테마 일괄 제어
